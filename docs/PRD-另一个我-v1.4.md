# 「另一个我」— 产品需求文档（PRD）

> **版本：** v1.4
> **日期：** 2026-04-09
> **状态：** 逻辑增强版（新增锚定时间线、戏剧化实现细节、情绪回归测试、架构细化、模型管理器）

---

## 1. 产品概述

### 1.1 产品名称

**中文名：** 另一个我
**英文名：** Another Me
**产品定位：** 本地化人生决策推演引擎 —— 基于用户真实画像，模拟"如果当初选了另一条路"的人生轨迹。

### 1.2 一句话描述

> 每个人生选择的背后，都有一条被放弃的时间线。「另一个我」帮你看见它。

### 1.3 核心价值

| 价值点 | 描述 |
|--------|------|
| **自我认知** | 通过推演更了解自己的真实渴望和恐惧 |
| **决策辅助** | 不给答案，但提供更全面的视角 |
| **情感出口** | 说出来、写下来，让纠结有个安放之处 |
| **隐私安全** | 完全本地运行，敞开心扉无需顾虑 |

### 1.4 目标用户

- **主要用户：** 18-40岁中文互联网用户
- **次要用户：** 海外华人（双语支持）
- **使用场景：** 无聊时随便看看、重大人生抉择前的思考

## 2. 用户画像系统

### 2.1 画像维度

#### 必填项（影响推演核心逻辑）

| 维度 | 选项示例 | 用途 |
|------|---------|------|
| **职业** | 学生 / 程序员 / 设计师 / 自由职业 / 待业 / 创业者 / ... | 决定收入曲线、社会关系 |
| **日常习惯** | 游戏 / 看剧 / 刷社交媒体 / 阅读 / 健身 / 撸铁 / 听播客 / ... | 决定「业余时间」行为预测 |
| **社交倾向** | 社恐 / 偏内向 / 中性 / 偏外向 / 社交达人 | 决定人际关系走向 |
| **经济状况** | 月光 / 略有积蓄 / 有存款 / 背负贷款 / 经济独立 | 决定财务推演可信度 |
| **性格标签** | 拖延症 / 行动派 / 完美主义 / 随遇而安 / 焦虑型 / ... | 决定决策执行质量 |
| **感情状态** | 单身 / 暗恋中 / 恋爱中 / 已婚 / 离异 / ... | 决定感情线推演 |

#### 选填项（让推演更精准）

| 维度 | 说明 |
|------|------|
| 健康状况 | 影响精力、情绪等长期变量 |
| 家庭背景 | 影响人生选择和责任感 |
| 地理位置 | 影响职业机会、社交圈 |
| 核心恐惧 | 决定人生路径的情感驱动力 |
| 梦想/目标 | 影响长期满足感计算 |

### 2.2 画像录入流程

**首次启动引导（Onboarding）：**

```
Step 1: 欢迎页
        「在开始之前，我想了解一下你」
        [开始了解]

Step 2: 基础信息（职业、感情状态、经济状况）
        → 单选卡片式交互

Step 3: 日常习惯（可多选 + 自由补充）
        → 「你平时下班/放学后一般做什么？」

Step 4: 性格自评
        → 「你觉得自己是哪种人？」
        → 拖动条：内向 ←————————→ 外向
        → 拖动条：冲动 ←————————→ 谨慎
        → 拖动条：悲观 ←————————→ 乐观

Step 5: 完成
        「记住你了。我们开始吧。」
```

### 2.3 画像动态修正机制

**核心原则：人会说谎，人会变化。**

```
每次推演时：
┌─────────────────────────────────────────┐
│ 「最近有什么变化吗？」                    │
│                                         │
│ [ ] 换了工作   [ ] 搬家了               │
│ [ ] 谈恋爱了   [ ] 开始健身了            │
│ [ ] 减了肥     [ ] 考证了               │
│ [ ] 其他（自由输入）                    │
│                                         │
│ 或直接用上次的信息继续 →                 │
└─────────────────────────────────────────┘

LLM 自动检测：
- 用户历史输入中反复出现的模式 → 修正画像标签
- 用户新输入中暴露的新习惯 → 提示用户确认是否更新
- 画像变更需要用户显式确认，不可静默修改
```

### 2.4 隐性画像（不展示给用户）

LLM 通过对话自动推断，不展示但影响推演：

| 推断标签 | 来源 | 影响 |
|---------|------|------|
| 意志力强弱 | 多次输入"想减肥但没动" | 影响习惯改变的推演 |
| 决策困难症 | 输入时反复纠结措辞 | 影响推演中"犹豫"的描写 |
| 职场困扰 | 多次提到"领导/同事" | 影响职业线情感色彩 |
| 家庭压力 | 提及"父母/催婚/房贷" | 影响生活线权重 |

## 3. 核心功能

### 3.1 蝴蝶效应模拟器

**功能描述：** 输入一个微小决定，LLM 基于用户画像推演 5 次，每次叠加随机扰动，最终归纳为 3 条时间线。用户可调节戏剧化程度。

### 3.1.0 戏剧化程度调节器（Drama Level）

> **新增。** 让用户自己决定想看到多少「戏剧性」的人生推演。

#### 3.1.1 调节档位

| 档位 | 名称 | 概率分布 | 描述 |
|------|------|---------|------|
| 1 | **普通人的故事** | 90%普通 + 10%小波动 | 绝大多数时间线都是普通人生，偶尔有点小涟漪 |
| 2 | **适度的戏剧** | 60%普通 + 30%波动 + 10%意外 | 大多数是普通人生，但有一些让人心动的变化 |
| 3 | **命运的岔路** | 30%普通 + 40%波动 + 30%戏剧 | 一半时间线有「大起大落」，让你意识到选择的重量 |
| 4 | **平行宇宙狂热** | 0%普通 + 50%戏剧 + 50%极端 | 每条时间线都有截然不同的命运，适合好奇心驱动用户 |

#### 3.1.2 界面设计

│  「你想看到怎样的人生可能性？」                            │

│     ↓                                                   │

│  ┌─────────────────────────────────────────────────┐   │
│  │  90% 的推演结果会是普通人生                       │   │
│  │  10% 会有一些小波折，但总体平稳                   │   │
│  │                                                   │   │

#### 3.1.3 各档位详细说明

**档位 1：普通人的故事（默认推荐）**

```
时间线分布：
- 90%：「稳定普通型」：工作、恋爱、结婚、生子、退休，平凡但真实
- 10%：「小波折型」：失业后重新就业、恋爱分手后复合、小病一场后康复

核心叙事主题：
「人生没有那么多大起大落，但每一个小选择都在悄悄改变生活的质地。」
```

**档位 2：适度的戏剧**

```
时间线分布：
- 50%：「稳定普通型」
- 30%：「意外收获型」：跳槽后意外发现新方向、单身时意外遇到真爱
- 20%：「小遗憾型」：错过了一些机会，但得到了其他东西

核心叙事主题：
「有些选择会带你走向不同的风景，但大多数风景都值得一看。」
```

**档位 3：命运的岔路**

```
时间线分布：
- 30%：「稳定普通型」
- 40%：「大起大落型」：创业成功/失败、婚姻幸福/破裂、事业腾飞/停滞
- 30%：「意外转折型」：大病一场后重新审视人生、偶遇贵人改变命运

核心叙事主题：
「每一个重大选择都在重新定义你的人生边界。」
```

**档位 4：平行宇宙狂热**

```
时间线分布：
- 100%：「极端分叉」：每条时间线都有截然不同的命运走向
  - 时间线A：从月薪1万到财富自由
  - 时间线B：从健康到重病缠身
  - 时间线C：从单身到三个孩子
  - 时间线D：从社恐到社交达人
  - 时间线E：各种极端命运的组合

核心叙事主题：
「如果你想看看所有可能性，这就是属于你的平行宇宙。」
```

#### 3.1.4 Prompt 约束注入

```
【戏剧化程度约束注入示例 - 档位1】

请严格按照以下概率分布生成推演结果：
- 90% 的推演应该描述「普通但真实」的人生轨迹
- 10% 可以有小的波折，但不要有毁灭性的结局
- 避免极端化的描述（如：猝死/一夜暴富/彻底改变命运）
- 重点描述日常生活的小确幸和小遗憾
- 「普通」不等于「失败」，平稳的生活本身就有价值

【戏剧化程度约束注入示例 - 档位4】

请生成极具戏剧性的推演结果：
- 每条时间线必须有「命运性」的重大转折
- 可以包含：创业暴富/破产、婚姻幸福/破裂、重大疾病/意外康复
- 要让用户感受到「不同的选择真的带来了完全不同的人生」
- 叙事要有张力，描写要具体生动
```

#### 3.1.5 特殊处理：「普通」的尊严

> **核心原则：** 无论哪个档位，都不能让「普通」听起来像是一种失败。

```
✗ 错误示范（档位1）：
「如果你选择了B，你将过上普通的生活，每月工资刚好够花，
 没有太多惊喜，也没有太多波澜。你会在平凡中度过一生。」

✓ 正确示范（档位1）：
「如果你选择了B，你会过上踏实的生活。有稳定的收入，
 和爱的人一起吃饭，周末和老朋友聚会。孩子慢慢长大，
 父母慢慢老去。你会经历一些小小的焦虑，也会收获很多小确幸。
 这就是真实的人生。」

✓ 正确示范（档位4 - 极端对比）：
「时间线A：你抓住了那次机会，30岁实现了财务自由，开始环游世界...
 时间线B：你错过了那次机会，30岁还在原来的岗位，但你在这个过程中
 成为了一个好丈夫/好妻子，孩子叫你『最棒的爸爸/妈妈』...
 （两条线都真实，都有各自的意义）」
```

#### 3.1.6 默认值建议

```
首次用户：默认档位 1（普通人的故事）
- 减少首次使用时的认知负担和情感压力
- 建立「真实感」信任

老用户（5次以上推演）：可以解锁更高档位
- 用户已经理解产品逻辑，好奇心驱动探索更戏剧化的结果

用户可以随时在设置中修改默认档位
```

#### 3.1.7 戏剧化程度 → Temperature 实现映射

> **工程实现贴士。** 将四档戏剧化程度直接映射为 Ollama 的 `temperature` 参数值，控制 LLM输出的随机性。

| 档位 | 名称 | Temperature 范围 | 效果描述 |
|------|------|-----------------|---------|
| 1 | 普通人的故事 | 0.3 ~ 0.5 | 保守、可预测，LLM 倾向选择最"合理"的叙事 |
| 2 | 适度的戏剧 | 0.5 ~ 0.7 | 适度随机，出现合理的意外转折 |
| 3 | 命运的岔路 | 0.7 ~ 0.9 | 较高随机性，重大抉择更果断、更具戏剧张力 |
| 4 | 平行宇宙狂热 | 0.9 ~ 1.2 | 最高随机性，允许离谱但有趣的极端推演 |

**Rust 侧实现示例：**

```rust
fn drama_to_temperature(drama_level: u8) -> f32 {
    match drama_level {
        1 => 0.3 + rand::random::<f32>() * 0.2,  // 0.3~0.5
        2 => 0.5 + rand::random::<f32>() * 0.2,  // 0.5~0.7
        3 => 0.7 + rand::random::<f32>() * 0.2,  // 0.7~0.9
        4 => 0.9 + rand::random::<f32>() * 0.3,  // 0.9~1.2
        _ => 0.5,
    }
}
```

#### 3.1.8 黑天鹅因子开关（Black Swan Toggle）

> **独立于戏剧档位。** 即使在"普通人的故事"档位，用户也可开启此开关，获得极低概率的"命运性随机事件"。

**设计规格：**

| 开关状态 | 描述 | 触发概率 |
|---------|------|---------|
| 关闭（默认） | 只受档位概率影响，不触发黑天鹅 | 0% |
| 开启 | 任何档位下，均有 **3% 概率**触发一次彻底改变人生的随机事件 | 3% |

**黑天鹅事件库：**

```python
black_swan_events = [
    # 负面黑天鹅
    "突发重病，经历生死考验后彻底改变人生优先级",
    "家人突发意外，被迫离开原有城市/职业轨道",
    "金融危机中储蓄大幅缩水，被迫重新规划财务",
    # 正面黑天鹅
    "偶然遇到贵人，意外获得重大机会",
    "买彩票/投资意外获得超额回报",
    "旅行中遇到灵魂伴侣，彻底改变人生方向",
    # 中性黑天鹅
    "偶然看到一本书/视频，人生观被彻底颠覆",
]
```

**交互设计：**

```
┌─────────────────────────────────────────────────────────┐
│  🌀 黑天鹅因子                                          │
│                                                         │
│  [ ] 开启黑天鹅事件（即使在「普通」档位也有极低概率      │
│      触发彻底改变人生的随机事件）                         │
│                                                         │
│  ℹ️ 开启后：                                           │
│     • 3% 概率触发一次"命运性"随机事件                   │
│     • 事件从正面/负面/中性事件库中随机抽取               │
│     • 不会连续触发（单次推演最多1次）                    │
└─────────────────────────────────────────────────────────┘
```

#### 3.1.1 推演逻辑

```
输入：「今天我没去参加聚会」

第一次推演（基线）：
  ↓ LLM 推理 + 用户画像约束
输出A₁：「你没去聚会，晚上在家打游戏，意外在游戏论坛认识了一个外包客户...」

第二次推演（基线 + 扰动）：
  输入：输出A₁ + 随机扰动（luck_factor ±15%, health_var ±10%...）
  ↓
输出A₂：「你没去聚会，本想打游戏，但突然想看点东西，于是刷到了一个课程广告...」

...（共5次）

聚类分析 → 归纳为3种「典型人生路径」：
  路径1：稳健型（稳步成长→小波折→平稳人生）
  路径2：转折型（意外机会→方向转变→新的可能）
  路径3：极端型（大起大落→黑天鹅事件→命运性转折）
```

#### 3.1.2 扰动因子库（Rust 实现）

| 因子 | 类型 | 扰动范围 |
|------|------|---------|
| `luck_factor` | 连续值 | ±15% |
| `health_var` | 连续值 | ±10% |
| `relationship_miracle` | 布尔值 | 偶发 |
| `market_condition` | 连续值 | ±20%（时代红利） |
| `personal_breakdown` | 布尔值 | 偶发 |
| `black_swan` | 布尔值 | 5% 概率触发 |
| **行为习惯偏移** | 连续值 | ±30%（打游戏→可能看书） |

#### 3.1.3 习惯偏移的特殊处理

> **核心洞察：** 没有人 100% 做某事。打游戏的人也可能某天突然想看书。

```
「蝴蝶效应的扰动，必须基于用户画像但不完全囿于用户画像」

推演时：
1. 基线行为：70% 概率按用户画像行动
2. 偏移概率：25% 概率出现画像外的行为
3. 极端偏移：5% 概率出现完全相反的行为

示例（用户画像：每天打游戏4小时，从不看书的游戏玩家）：
- 基线：75% 在家打游戏
- 偏移：20% 打游戏打到一半想换换口味，刷到了某个视频/文章
- 极端：5% 完全不打游戏，选择学习/运动/社交
```

### 3.2 未来自我的来信

**功能描述：** 不输出干巴巴的数据图表，而是以「10 年后你」的口吻写一封信。信件的语气由「情绪维度」决定。

#### 3.2.1 情绪维度系统

```python
emotion_dimensions = {
    "energy":      0-100,   # 活力值（高=精力充沛，低=疲惫）
    "satisfaction": 0-100,  # 满足感（对人生选择的无悔程度）
    "regret":      0-100,   # 遗憾度（对未选择的路的好奇）
    "hope":        0-100,   # 希望值（对未来的期待）
    "loneliness":  0-100    # 孤独感（人际关系质量）
}
```

#### 3.2.2 情绪 → 语气映射

| 组合特征 | 语气类型 | 信件开头示例 |
|---------|---------|------------|
| 高能量 + 高满足 | 温暖鼓励型 | "嘿，我是十年后的你。一切都在变得更好..." |
| 低能量 + 高遗憾 | 疲惫感慨型 | "我不知道怎么开口，但我需要告诉你..." |
| 高孤独 + 低希望 | 沉默疏离型 | "你应该不会想知道十年后发生了什么..." |
| 高希望 + 低遗憾 | 平静叙述型 | "给你写这封信，想说说我的想法..." |
| 混乱型 | 黑色幽默型 | "如果有人告诉你十年后你会变成这样，你肯定会..." |

#### 3.2.3 信件结构模板

```
第一段：建立身份
  「我是十年后的你。写这封信的时候，我正坐在[场景描述]...」

第二段：回应你的当下
  「我知道你现在在纠结[用户的决策]...」

第三段：如果选 A（现实选择）
  「如果你选了[用户的选择]...」

第四段：如果选 B（平行选择）
  「但如果你没选...」

第五段：给现在的你的建议

## 3.4 记忆连贯性（Temporal Consistency）

> **新增核心机制。** 让推演不再是孤立的离散事件，而是构建一条「完整的数字平行人生」。

### 3.4.1 核心概念

```
传统模式（孤立推演）：
  推演1：「如果半年前我辞职」
  推演2：「如果现在我结婚」
  → 两次推演独立，互不知晓

记忆连贯模式（因果链）：
  推演1：「如果半年前我辞职」→ 结论：创业成功，性格变得更有侵略性
  推演2：「如果现在我结婚」→ 自动引入推演1的影响：事业型人格 + 家庭责任 = 更保守？
  → 推演之间形成因果链条
```

### 3.4.2 因果链检索机制

每次新推演时，系统自动检索历史决策库，构建「人生背景上下文」：

```typescript
interface LifeContext {
  // 历史推演摘要
  past_decisions: PastDecisionSummary[];

  // 推断出的性格演变
  personality_evolution: {
    before: string[];    // 半年前的性格标签
    after: string[];     // 当前的性格标签
    change_reason: string; // 变化原因
  };

  // 当前人生状态
  current_state: {
    career_stage: string;
    relationship_status: string;
    financial_health: string;
    health_trend: string; // 上升/平稳/下降
  };
}

interface PastDecisionSummary {
  decision_id: string;
  decision_text: string;
  simulated_date: string;       // 什么时候做的这个推演

  personality_impact: string[];  // 对性格的影响
  timeline_type: 'reality' | 'parallel';
}
```

### 3.4.5 锚定时间线（Primary Timeline）

> **解决"祖父悖论"。** 当用户在多个时间线上做过推演，新推演应继承哪条线的性格演变？引入用户主动标记机制，消除歧义。

#### 问题场景

```
用户在 4 月推演了「跳槽 A 线」→ 结论：性格变得更有侵略性
用户在 5 月推演了「留在 B 公司线」→ 结论：性格维持保守稳健
用户在 6 月推演「是否结婚」→ 此时性格应该是「侵略性」还是「稳健」？
```

#### 解决方案：锚定时间线

```typescript
interface AnchorTimeline {
  is_anchored: boolean;         // 用户是否将该线标记为锚定
  anchored_timeline_id?: string; // 锚定的具体时间线ID
  anchored_reason?: string;     // 锚定理由（用户填写，可选）
  set_at: string;               // 锚定时间
}
```

**交互设计：**

```
┌─────────────────────────────────────────────────────────┐
│  📌 哪条时间线最接近你想要的未来？                         │
│                                                         │
│  [推演A：跳槽去A公司]  ← 当前锚定线（你标记的）           │
│  [推演B：留在原公司]                                     │
│                                                         │
│  「后续推演将基于跳槽A线展开」                             │
│                                                         │
│  [更换锚定线]  [取消锚定，系统自动推断]                     │
└─────────────────────────────────────────────────────────┘
```

**规则：**
- 用户可将任意历史推演标记为"锚定时间线"
- 锚定后，新推演优先继承该线的性格演变和人生状态
- 最多同时存在 **1 条活跃锚定线**（可随时切换）
- 用户也可选择"取消锚定"，让系统根据因果链自动推断最合理的背景
- 锚定切换不影响已生成的推演记录，只影响新推演的上下文

## 3.5 伦理与安全阀（Safety Valve）

> **新增。** 处理「人生决策」和「后悔」是非常沉重的情感话题，需要内置安全机制。

### 3.5.1 核心风险识别

| 风险场景 | 潜在危害 |
|---------|---------|
| 平行人生过于完美 | 加剧用户对现实的不满，觉得「现在的自己很失败」 |
| 推演结果过于负面 | 可能引发用户的焦虑或抑郁情绪 |
| 用户沉迷推演 | 用「另一个自己」逃避现实决策 |
| 信息过载 | 3条时间线 + 决策树 + 来信，信息量太大造成心理负担 |

### 3.5.2 「现实连接」机制

**强制在每封「未来自我的来信」末尾加入「当下闪光点」：**

```
无论信件内容是正面的还是负面的，强制在最后一段添加：

──────────────────────────────────────────
📌 来自未来的提醒：

虽然我在推演这些可能性，但别忘了——
2026年的你，依然拥有：

• {用户画像中的某项优势}
• {基于历史推演推断出的成长}
• {当前正在发生的好事}

「你现在的每一天，都是未来那个自己回不去的曾经。」

—— 十年后的你
──────────────────────────────────────────
```

**闪光点来源（LLM 自动提取）：**

```python
def generate_shine_points(user_profile: UserProfile, recent_history: list) -> list:
    """生成「当下闪光点」列表"""
    shine_points = []

    # 1. 从用户画像中提取优势
    if user_profile.occupation == "学生":
        shine_points.append("年轻拥有的可能性")
    if user_profile.health_status == "健康":
        shine_points.append("健康的身体")
    if "健身" in user_profile.habits:
        shine_points.append("对自我提升的坚持")

    # 2. 从历史推演中提取已实现的可能性
    for record in recent_history:
        if record.type == "reality" and record.outcome_positive:
            shine_points.append(f"你曾经选择：{record.decision_text}，结果是好的")

    # 3. 当前时刻的美好（动态生成）
    shine_points.append("此刻正在阅读这个故事的专注力")

    return shine_points[:3]  # 最多3条
```

### 3.5.3 正向引导原则

**Prompt 中内置引导指令：**

```
【正向引导原则】
在描述任何时间线时，遵循以下原则：

1. 避免「选错了毁一生」的叙事框架
   ✗ 错误：「如果你当初没辞职，你现在还在温水中等死」
   ✓ 正确：「如果你没辞职，你可能会错过一些机会，但也保留了另一种安稳」

2. 强调「每条路都有风景」
   ✗ 错误：「平行的你比现实的你成功太多」
   ✓ 正确：「那条路上的风景不同，但同样真实」

3. 给予用户自主权
   「最终，你才是自己人生的作者。这些推演只是帮助你看到可能性。」

4. 设置「推演上限」提醒
   如果用户连续推演超过3次/天，弹出善意提醒：
   「你已经推演了3次。有时候，最好的决定是活在当下。」

5. 黑暗内容预警
   如果推演中出现「重病/离婚/破产」等黑暗内容，
   在展示前弹出：「以下内容可能引起不适，是否继续？」

### 3.5.4 情绪回归测试（Emotional Recovery Test）

> **主动干预机制。** 当 LLM 生成的叙事被判定为"极度负面"时，系统强制注入转机描述，防止用户因推演结果产生现实抑郁。

**触发条件（同时满足）：**

```python
def is_extremely_negative(emotion_dims: EmotionDimensions) -> bool:
    """五维中有任意三维同时低于20分"""
    low_dims = sum(1 for v in emotion_dims.values() if v < 20)
    return low_dims >= 3
```

**触发后的处理流程：**

```
Step 1: 现实主义引擎检测到「极度负面」
  → 暂停直接展示给用户

Step 2: LLM 被要求在叙事末尾强制追加「转机描述」
  ┌─────────────────────────────────────────────────────────┐
  │ 📌 [系统注入 - 转机描述]                                │
  │                                                        │
  │ 「......（原叙事中的低谷期）」                           │
  │                                                        │
  │ 「但正是在最艰难的时刻，你开始意识到……」                  │
  │ 「那些你以为失去的东西，其实以另一种形式回来了。           │
  │   你比自己想象中更有韧性。」                            │
  └─────────────────────────────────────────────────────────┘

Step 3: 弹出「心理健康小贴士」（可关闭）
  ┌─────────────────────────────────────────────────────────┐
  │ 💙 来自「另一个我」的关心                               │
  │                                                        │
  │ 「推演的结果只是可能性，不是预言。                       │
  │   如果这些内容让你感到不适，你可以通过 [反馈按钮]          │
  │   告诉我们，也可以随时休息一下。」                        │
  │                                                        │
  │ [我知道了]  [提供更多帮助资源]                           │
  └─────────────────────────────────────────────────────────┘
```

**心理健康资源示例（本地存储，不联网）：**

```
- 「如果感到持续的沮丧或焦虑，建议寻求专业心理支持」
- 「全国心理援助热线：400-161-9995（24小时）」
- 「记住：感到不确定和迷茫，是每个人都会经历的人生常态」
```

**设计原则：**
- 强制注入的「转机描述」由 LLM 基于原叙事自然生成，不是模板套话
- 「心理健康小贴士」仅在极度负面时出现，不每次推演都弹
- 用户可随时在设置中关闭此功能（但关闭时会在推演前提示）
- 心理健康资源为纯本地文本，不涉及任何网络请求

## 3.7 音频化「未来信件」（Voice Clone）

> **新增（Phase 3）。** 利用本地 TTS 模型，让「未来自我的来信」用用户自己的声音朗读出来。

### 3.7.1 核心价值

### 4.5 现实主义因子

```python
class RealismEngine:
    """
    确保推演结果既不过于乐观，也不过于悲观。
    """

    MUST_INCLUDE = {
        "ups": (2, 3),        # 每个版本必须有2-3个高光时刻
        "downs": (1, 2),      # 必须有1-2个低谷
        "accidents": (0, 1),  # 至少一条线要有意外事件
        "surprises": (1, 1)  # 至少一个意想不到的转折
    }

    POSITIVITY_RATIO_RANGE = (0.4, 0.6)  # 积极:消极 应该在 4:6 到 6:4 之间

    def check_realism(self, timeline: Timeline) -> RealismCheckResult:
        """
        检查单条时间线的现实主义程度
        """
        positivity = count_positive_sentiment(timeline.narrative)
        negativity = count_negative_sentiment(timeline.narrative)
        ratio = positivity / (positivity + negativity + 0.001)

        if ratio > self.POSITIVITY_RATIO_RANGE[1]:
            return RealismCheckResult(
                status="TOO_POSITIVE",
                suggestion="需要注入挫折和低谷"
            )
        elif ratio < self.POSITIVITY_RATIO_RANGE[0]:
            return RealismCheckResult(
                status="TOO_NEGATIVE",
                suggestion="需要注入希望和高光时刻"
            )
        else:
            return RealismCheckResult(
                status="BALANCED",
                suggestion=None
            )

    def apply_realism_correction(self, timeline: Timeline) -> Timeline:
        """
        如果现实主义检查失败，注入修正内容
        """
        check = self.check_realism(timeline)
        if check.status == "BALANCED":
            return timeline

        # 调用 LLM 重新生成，注入现实主义修正指令
        correction_prompt = f"""
        当前推演过于{check.status.replace('TOO_', '')}，
        {check.suggestion}。
        请重新生成...
        """
        return llm.regenerate(timeline, correction_prompt)
```

## 9. 风险与挑战

| 风险 | 等级 | 应对策略 |
|------|------|---------|
| 本地模型推理速度慢 | 中 | 提供「快速模式」选项（牺牲质量换速度） |
| 推演结果过于相似 | 中 | 优化扰动因子库，增加多样性 |
| LLM 幻觉导致荒谬推演 | 高 | 强化 Prompt 约束 + 现实主义因子双重校验 |
| 用户对「未选的路」产生执念 | 中 | 安全阀机制 + 正向引导原则 |
| 中英翻译质量参差 | 低 | 优先中文，英文作为辅助 |
| **平行人生过于完美引发焦虑** | **高** | **安全阀「当下闪光点」强制注入** |
| **用户沉迷推演逃避现实** | **中** | **推演上限提醒（3次/天）+ 正向引导** |
| **因果链推理复杂度指数增长** | **中** | **限制因果链深度（最多5个历史决策）** |
| **音频克隆技术门槛高** | **中** | **Phase 3实现，提供降级方案（文字/AI播音员）** |
| **反馈数据不足导致画像进化慢** | **低** | **引导用户反馈，降低反馈门槛** |
